# セキュリティルール

## 秘密情報
- APIキー、パスワード、トークンをソースコードにハードコードするな
- .envファイルをコミットするな
- 秘密情報は環境変数で管理
- ログに機密データを出力するな

## 入力検証
- システム境界で全ての入力をバリデーション
- SQLは必ずパラメタライズドクエリ
- HTML出力はエスケープ（XSS防止）

## 保護ディレクトリ
以下のディレクトリの変更には明示的なユーザー承認が必要:
- `src/auth/` — 認証・認可ロジック
- `src/billing/` — 決済・金融データ
- `migrations/` — DBスキーマ変更
